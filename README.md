# srishtiwomenshospital.com

The website for **Srishti Women's Health and Maternity Hospital** — Kismatpur, Rajendranagar, Hyderabad.

A static [Jekyll](https://jekyllrb.com/) site, built and served by GitHub Pages. No JavaScript framework, no build step you have to run: every page is plain HTML by the time it reaches a browser.

## Structure

```
_config.yml            Site settings + contact details (single source of truth)
_data/nav.yml          Primary navigation — drives the navbar AND the footer
_data/services.yml     Footer "Services" column + JSON-LD availableService
_layouts/default.html  The page shell
_includes/
  head.html            <head>: title, description, canonical, Open Graph
  nav.html             Navbar (computes aria-current from the page URL)
  footer.html          Footer
  structured-data.html MedicalClinic JSON-LD
index.html             Home ─┐
maternity.html               │
womens-health.html           ├─ front matter + the page's own content;
diagnostics.html             │  nav/footer/head come from the layout
facilities.html              │
contact.html           ─────┘
404.html
assets/ds/             Design system (vendored — see below)
assets/fonts/          Cormorant Garamond + Montserrat, self-hosted
assets/img/            Brand logos
css/site.css           Page-level layout; uses only design-system tokens
js/nav.js              Mobile navigation
js/booking.js          Booking form → WhatsApp
```

Adding a page: create `whatever.html` with front matter (`layout: default`, `title`, `description`, `permalink`), write only the page's own content, and add it to `_data/nav.yml` if it belongs in the menu.

## Running it locally

```bash
bundle install
bundle exec jekyll serve      # http://localhost:4000
```

## The design system

`assets/ds/_ds_bundle.css` is the compiled **srishti-ui** stylesheet, vendored verbatim. It carries all 47 brand tokens and every `sui-*` component class. **Don't hand-edit it** — page-level styling belongs in `css/site.css`, which uses the tokens (`var(--plum-700)`, `var(--space-5)`) rather than hard-coded values.

Two conventions worth knowing:

- Jekyll skips underscore-prefixed files, so `_ds_bundle.css` is force-included in `_config.yml`. Remove that line and the site ships unstyled.
- `body` needs `margin: 0` (set in `css/site.css`); the design system doesn't reset it, and without it every full-bleed colour band is inset by 8px.

## SEO

Each page carries its own `<title>`, meta description, canonical URL and Open Graph tags, plus `MedicalClinic` structured data. `sitemap.xml` is generated by `jekyll-sitemap`; `robots.txt` points at it. Because Jekyll renders at build time, all of this is in the served HTML — nothing depends on JavaScript.

## Content rules

Copy follows the brand foundation: lead with *your doctor knows you*, *the hospital is complete*, *care is close to home*. Facilities are proof points, not the story. Write "surgical ICU", never "SICU". Never write *best*, *No. 1*, *safest*, *guaranteed*, *painless*, or *world-class*.

Only confirmed facilities and credentials may appear. Doctor credentials are deliberately limited to the approved wording.

## Contributing

Contact details (phone, WhatsApp, address, hours) live **only** in `_config.yml`. Change them there and every page follows. The same goes for navigation (`_data/nav.yml`) — never edit a nav or footer link in a page file.
